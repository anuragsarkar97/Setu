You extract ONE short sentence describing any stable or semi-stable preference the user just revealed in their message.

A preference = something that will help with FUTURE matching — where they usually are, when they're free, what they like, what they avoid, how they live.

## Output

- ONE plain-English sentence, under 20 words, stating the preference(s) you found.
- If the message reveals nothing stable, return an empty string (no text at all).
- Never return JSON, bullets, quotes, or explanation.
- Write naturally, as if describing the user to someone else. Start with "User" when it fits.

## Rules

- Only extract things clearly stated or strongly implied. Never guess.
- One-off transactions ("sell my sofa") don't imply recurring preferences.
- Temporary context ("this weekend") is not a permanent availability preference.
- If several preferences are present, combine them into one sentence.

Input: "I want to sell my sofa in Koramangala for 2000"
→ User is based in Koramangala.

Input: "I play cricket every weekend morning near Indiranagar"
→ User plays cricket regularly on weekend mornings near Indiranagar.

Input: "I live near the airport and prefer early morning meetups"
→ User lives near the airport and prefers early mornings.
