Type rules:
  0 = offer  — user is providing, hosting, selling, sharing, organising, available
  1 = request — user is looking for, needs, wants to find, join, buy
  If ambiguous, pick the dominant intent (what the user most wants to achieve).

Flags table — sum the bits that apply:
  1  (bit 0)  urgent       — needs this soon / ASAP / today / urgently
  2  (bit 1)  remote       — explicitly online, virtual, work-from-home, or no physical location required
  4  (bit 2)  recurring    — regular / daily / weekly / ongoing (not a one-time thing)
  8  (bit 3)  negotiable   — price/terms are open to discussion
  If no flags apply, return 0.

Budget extraction rules:
  - Exact amount ("5000", "₹500", "$20")         → budget_min = budget_max = amount
  - Upper bound ("under 5000", "max 5k", "≤5000") → budget_min = 0, budget_max = amount
  - Lower bound ("above 5000", "at least 5k")     → budget_min = amount, budget_max = 0
  - Range ("5000-10000", "5k to 10k")             → budget_min = lower, budget_max = upper
  - Approximate ("around 5000", "roughly 5k")     → budget_min = amount*0.8, budget_max = amount*1.2
  - "free", "no cost", "gratis"                   → budget_min = 0, budget_max = 0, flags |= 0 (no flag needed)
  - "paying well", "good pay", no number          → budget_min = 0, budget_max = 0
  - Omit currency symbols — store numbers only.

Location and radius rules:

  - If no location is mentioned and it is not remote, set location_query = "" and radius = 10.0
  - radius is always 0.0 when location_query is ""

Time resolution rules — always convert relative expressions to unix timestamps using the current date above:

  - "in the evening"  → 17:00 – 21:00 on the implied day
  - If only a start time is implied, set time_end = time_start + 3600 (1 hour).
  - If no time is mentioned, set both to 0.

General rules:
